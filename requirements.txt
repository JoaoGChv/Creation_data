Para realizar a instalação definitiva do **pyrealsense2** e do SDK no seu ambiente (Ubuntu 24.04 com a D415), o processo deve ser dividido em duas partes: a instalação das bibliotecas de sistema (drivers) e o wrapper de Python.

Aqui está o roteiro consolidado que funcionou no seu laboratório:

### 1. Instalação do SDK (Bibliotecas de Sistema)

No Ubuntu 24.04 (Noble), é necessário registrar os repositórios oficiais da Intel para garantir que o firmware da câmera seja reconhecido corretamente.

* **Registrar a chave de segurança:**
```bash
sudo mkdir -p /etc/apt/keyrings
curl -sSf https://librealsense.intel.com/Debian/librealsense.pgp | sudo tee /etc/apt/keyrings/librealsense.pgp > /dev/null

```


* **Adicionar o repositório Noble:**
```bash
echo "deb [signed-by=/etc/apt/keyrings/librealsense.pgp] https://librealsense.intel.com/Debian/apt-repo noble main" | sudo tee /etc/apt/sources.list.d/librealsense.list

```


* **Instalar os pacotes:**
```bash
sudo apt-get update
sudo apt-get install librealsense2-dkms librealsense2-utils librealsense2-dev librealsense2-dbg

```


*Nota: Se o pacote `dkms` der erro de conflito com o driver de vídeo (como aconteceu anteriormente), a recomendação é reiniciar o computador com a câmera desconectada e rodar `sudo dpkg --configure -a`.*

### 2. Configuração de Permissões (udev rules)

Para que o seu usuário consiga acessar a câmera sem precisar de `sudo`, é fundamental instalar as regras de dispositivo:

```bash
# Dentro da pasta onde você clonou ou baixou o SDK, ou via comando direto:
sudo cp /lib/udev/rules.d/60-librealsense2-udev-rules.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules && sudo udevadm trigger

```

### 3. Instalação do Wrapper Python (pyrealsense2)

Esta parte deve ser feita dentro do seu ambiente virtual (`venv`) para evitar conflitos com o Python do sistema.

* **Ativar o ambiente:**
```bash
source venv/bin/activate

```


* **Instalar via PIP:**
```bash
pip install pyrealsense2

```



### 4. Validação da Instalação

Para garantir que "as coisas de fato" estão funcionando:

1. **Teste de Hardware:** Rode `realsense-viewer` no terminal. Ele deve abrir uma interface gráfica e mostrar a D415. Verifique se o ícone indica USB 3.0 ou 2.1.
2. **Teste de Código:** Rode o script `realsense_capture.py` que estabilizamos. Ele deve imprimir os parâmetros intrínsecos ($fx, fy \approx 608$) e abrir a janela de captura.

**Dica para o seu README:** Ao documentar essa parte, enfatize que para o **Ubuntu 24.04**, o uso de um cabo de alta qualidade é vital, pois o sistema é mais sensível a quedas de frames no barramento USB 2.1.
